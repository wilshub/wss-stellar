Server-Side Languages

What is server-side scripting? Server-side scripting used in web development is a technique where a web server employs code scripts in order to create a response to each client’s request to the different websites they choose. There are many principles involved in this type of scripting which include:

Web Server Languages There are a range of different types of programming languages which are run on servers rather than clients. These languages are mainly used for creating, reading, deleting and updating information for large databases such as user accounts and up-to-date inventories all happening before the client-side languages such as HTML, CSS and JS. Some of the most popular of these languages are PHP, Ruby On Rails/Rails, Node JS and ASP.net.

Functionality

Functionality Server-side scripting can offer a selection of many useful functionalities to the client or user. The process is where the server executes programs which are able to output scripts that reside on web servers and user-accessible interactive websites onto web pages. Examples of instances like this include the shopping cart button found in many shopping and product establishments such as Argos where the customer can save favourite items and reserve them without having to worry about them being out of stock by the time they get round to paying for it.

Features

Features There are also many versatile features server-side scripting can offer to a client’s website in order to make it more dynamic and up-to-date with the latest technology.

Benefits include: Interpreted scripts are supplied to the programmer, rather than compiled where they have to create the code from scratch. This is easier because all the coder has to do is just read the interpreted script without having to execute it and basically just copy the code and editing it to their own devises afterwards.

Drawbacks include: There is increased complexity of developing; performance of the system is affected; and because the code is interpreted rather than compiled, some sequences of the code may not be relevant or meet the developer’s needs.

Accessibility

Accessibility Server-side scripting also aims to ensure that many people can view, access and navigate on the websites made as possible. Also many accessibility features are supplied such as alternative text, resizable fonts, Screen readers’ support and adjustable fonts.

Client Side vs Server Side

  • Client Side

    Also known as the "frontend".

    The code is run before the HTML text instead of after.

    The code is made almost exclusively using Javascript (JS).

    Common client-side langauages include HTML, CSS and JS.

  • Server Side

    Also known as the "backend".

    The code is run on the computer after the page has loaded.

    All code is executed on the server.

    Common server-side langauages include PHP, Ruby On Rails/Rails and ASP.net.
